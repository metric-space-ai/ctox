The Right-Click Revolution: Software on Demand

Why fit your operations into rigid SaaS templates? In CTOX Business OS, the entire workspace is a self-tuning canvas. Right-click anywhere to execute semantic database mutations or live-patch the actual application code on the fly:

Right-Click to Adapt Data Need custom fields, qualification scoring, or custom columns? Right-click the data grid and tell the AI. The background daemon dynamically shifts the local database schema and populates the records.
Right-Click to Adapt the App Want a custom button, a tailored form, or a different view? Right-click the UI element, type your prompt, and the AI hot-patches the actual module's HTML and Javascript code in real-time.
100% Private Sync Mesh Replication is fully peer-to-peer (WebRTC) between browser IndexedDB and the daemon's local SQLite store. All operational data stays local, firewalled, and protected behind your residential firewall.
One-Click Rollback Safety Every single AI hot-patch is automatically sealed, SHA-256 hashed, and logged inside the daemon's database. If you don't like an update, revert to any previous version instantly in one click.

Business OS connectivity

The browser does not need a direct IP route to CTOX.

CTOX can be public, routed through ctox.dev, or private behind NAT. The browser only needs the static Business OS shell and a pairing config. Data moves through CTOX DB WebRTC replication between browser IndexedDB and the Rust daemon's SQLite store.

Core Guarantees

Control and safety, built into the daemon.

The Fluid AI Paradigm

Traditional static SaaS forces manual developer cycles for database schema updates and application modifications. In Business OS, the integrated AI edits the raw HTML/JS/CSS modules and structures local SQLite tables in-place instantly upon a simple right-click request.

Authoritative State Validation

The browser UI is a fluid canvas, but the underlying Rust daemon enforces absolute guard gates. Every transaction, queue task, and command execution is continuously validated against 12 formal transition gates and 5 core state invariants.

Forensic Process Mining

CTOX does not rely on trust. Every database mutation is monitored by SQLite triggers and recorded in an audit trail, enabling Petri and DFG process discovery, deadlock checks, conformance audits, and safety violation scans.

Built-in Rollback Safety

Every AI UI hot-patch is automatically versioned, hashed, and sealed inside the daemon. A single click from the App Store drawer rolls back any module's files and directory state to its precise baseline instantly.

Install

Install CTOX on the host that should carry the work.

Use the terminal path when you are already on the target host. Open the TUI to configure runtime, communication, tickets, and remote access. Use the desktop app when you want a guided setup for this machine or remote servers, and when you want to manage more than one CTOX instance from one place.

Terminal install

Use this when you are already on the server or SSH session that should run CTOX.

curl -fsSL https://raw.githubusercontent.com/metric-space-ai/ctox/main/install.sh | bash ctox ctox start ctox status
  1. Run the installer on the host that should own the work.
  2. Open ctox to configure model/runtime, communication, tickets, and remote access.
  3. Start the daemon and check status before handing it real work.

Desktop install and instance manager

The desktop app can prepare CTOX on this computer or on SSH hosts, connect to remote WebRTC hosts, and keep multiple local or remote instances in one registry.
