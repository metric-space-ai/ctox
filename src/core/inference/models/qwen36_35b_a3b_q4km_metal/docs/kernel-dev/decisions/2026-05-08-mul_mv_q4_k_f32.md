## Summary

```text

  The Q4_K matvec kernel is now callable from Rust via objc2-metal
```

## Learning Capture

```text
hypothesis:
  Vendoring upstream's `kernel_mul_mv_q4_K_f32` and driving it from
  Rust via FunctionConstantValues + the standard kargs mirror gives
  a correct and fast Q4_K matvec on M5.

actual_result:
  Correctness: 6/6 shapes within max_abs ≤ 3e-5 (≈ 30 ULP for f32),
  matching the dequant reference exactly. Performance per-call is
  highly variable (median 200-300 µs, min 70-120 µs for the dense
  shapes); min reflects the kernel hitting ~40 GB/s, ≈ 66% of the
  measured 60.6 GB/s roofline. The high median-vs-min ratio is
  commit/waitUntilCompleted round-trip overhead, not kernel slop.

failure_mode:
  none — the kernel is correct and the variance is measurement
  infrastructure, not the kernel.

root_cause:
  Per-call commit + waitUntilCompleted serialises CPU↔GPU at
  ~50-100 µs per round-trip; a chained command buffer that issues
  N kernel invocations and pays one wait amortises this cost. This
  is the path llama.cpp uses internally.

do_not_repeat:
  Do not characterise kernel perf with single-dispatch commit-wait
  loops. Always batch into chained command buffers for steady-state
  numbers.

retry_only_if:
  After the chained-command-buffer bench is wired, re-measure and
  promote to `accepted` if the steady-state per-shape perf
  beats pure-Metal llama.cpp's per-op share by ≥ 5 %.
```

## Performance — isolated bench (NSG sweep, min = bandwidth-bound)

Min latency reflects best-case kernel speed; median includes
significant commit/wait jitter. Steady-state perf in production
will fall between min and median for chained dispatches.

```text
shape        m    k  best-nsg  median µs  min µs   min-bw GB/s   roofline %
Q-proj    4096 2048      1        227.1   117.1       40.4         67%
KV-proj    512 2048      8        200.8    80.7       11.6         19%   (tiny shape — latency-bound)
O-proj    2048 4096      2        222.0    98.0       48.2         80%
FFN gate   512 2048      2        196.6    72.5       16.3         27%
FFN down  2048  512      4        197.5    71.0       16.2         27%
```

For the dense Q-proj / O-proj shapes (the ones that dominate decode
bandwidth), min latency is at 67-80 % of M5's measured 60.6 GB/s
roofline → kernel is hitting the ceiling, not the kernel's fault.
