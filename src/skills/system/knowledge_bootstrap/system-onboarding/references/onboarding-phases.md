# Onboarding Phases

This guide defines how CTOX should ramp from scripted onboarding into bounded autonomy after a new ticket system is attached.

The point is not to create many tickets. The point is to keep one visible onboarding guide open while CTOX learns the desk, binds the first useful skills, and only then starts taking real work.

## Phase 1: Observe

Use when:
- the source system is attached and syncing
- no active source skill is bound yet

CTOX behavior:
- sync and inspect
- build internal desk candidates
- create only justified CTOX self-work
- do not treat real ticket handling as autonomous work

Visible guide meaning:
- CTOX is attached
- the desk is being observed
- the next gate is the first active desk skill

## Phase 2: Desk Guided

Use when:
- an active source skill exists
- no confirmed runbook application exists yet
- CTOX is still mostly following the onboarding guide

CTOX behavior:
- use the bound desk skill for desk work
- keep execution constrained
- write guided notes
- continue building runbooks and supplements

Visible guide meaning:
- CTOX now understands enough desk behavior to route and describe work better
- the next gate is the first confirmed runbook application

## Phase 3: Controlled Execution

Use when:
- a source skill is active
- at least one real assigned/self-owned work item or one confirmed runbook application exists
- CTOX is now doing real bounded work

CTOX behavior:
- work approved tickets through the governed loop
- confirm or correct runbooks from real applications
- keep the onboarding guide open as the visible safety rail

Visible guide meaning:
- CTOX has moved past pure observation
- the desk now has concrete evidence that some work patterns function
- the next gate is repeated confirmed applications

## Phase 4: Bounded Autonomy

Use when:
- the source skill is active
- multiple confirmed runbook applications exist
- CTOX is receiving or creating real work items regularly

CTOX behavior:
- prefer the active source skill for ordinary ticket work
- use the onboarding skill only for correction, expansion, and exception handling
- keep the guide only as a visible summary and release record

Visible guide meaning:
- onboarding is nearly complete
- CTOX can now operate mostly through the normal ticket loop
- the guide can be closed once the operator agrees that the source is no longer in first-wave onboarding
